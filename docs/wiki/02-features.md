# SubTrack 产品需求（精简版）

## 用户痛点

- **订阅支出模糊** 用户不知道每月订阅总共花多少钱，缺乏整体支出概念。
- **忘记取消订阅** 不再使用的服务仍在持续扣费，造成不必要的浪费。
- **续费日期难记** 多个订阅的扣款日期分散，容易遗漏或忘记。
- **预算缺乏规划** 没有设定订阅预算的概念，支出失控。
- **多币种困扰** 不同订阅使用不同货币，难以统一计算总支出。

## 核心功能（按优先级）

### P0（MVP 必需）

- 订阅管理：新增、编辑、删除、列表、详情
- 预算管理：月预算设置、预算进度展示、月度支出计算
- 多币种：金额录入、基准货币设置、统一换算
    - 多货币输入 支持 CNY/USD/EUR/GBP/JPY/HKD/TWD
    - 基准货币设置 用户选择统计时的基准货币
    - 汇率转换 自动转换为基准货币统计
    - 原币种显示 同时显示原始货币和转换后金额
- 首页概览：总支出、预算进度、即将续费

### P1（首个云端版本）

- 订阅状态管理：取消/恢复、取消日期记录
- 可视化增强：分类占比、基础趋势统计
- 数据能力：导入导出、云端同步与冲突提示
- 基础提醒：小程序内即将续费提醒

### P2（后续迭代）

- 超支预警 微信订阅消息提醒
- 更长周期的趋势分析
- 高级统计
- 家庭共享、价格监控等增值功能


## 关键数据字段（订阅）


| 字段                        | 说明                              |
| ------------------------- | ------------------------------- |
| name                      | 订阅名称                            |
| amount                    | 费用（分）                           |
| currency                  | 货币（CNY/USD/EUR/GBP/JPY/HKD/TWD） |
| billingCycle              | monthly/quarterly/yearly        |
| billingDay / billingMonth | 扣款日（年付需月）                       |
| status                    | active/cancelled                |
| nextBillingDate           | 下次扣款时间                          |
| category / remark         | 分类与备注                           |


## 主要页面

- `首页`：预算卡片 + 即将续费 + 分类分布
- `订阅`：按状态查看、筛选、排序、进入详情
- `新增/编辑`：金额币种、周期、扣款日期、分类备注
- `统计`：周期趋势和分类统计
- `我的`：预算、汇率、数据导入导出

## 非功能要求

- 性能：首屏 < 3s，列表流畅
- 兼容：微信基础库 2.10+，iOS/Android
- 安全：HTTPS、敏感信息脱敏、不越权访问
- 可用性：离线可读、网络失败可重试
