# SubTrack 产品概述

## 1. 项目背景

SubTrack 是一款个人财务管理类小程序，专注于追踪和管理用户的软件订阅服务。随着订阅经济的兴起，用户订阅的服务越来越多，但缺乏有效的管理工具，导致支出不透明、忘记取消不需要的订阅等问题。

SubTrack 旨在帮助用户清晰了解每月/每年的订阅支出，并有效设定和控制预算。

## 2. 目标用户

### 2.1 核心用户画像

- **数字服务重度用户**：拥有 5+ 个软件/服务订阅
- **预算意识强**：希望管理订阅支出、控制预算
- **健忘型用户**：经常忘记取消订阅或忘记续费日期
- **多币种用户**：订阅服务涉及多种货币（CNY、USD、EUR 等）

### 2.2 用户场景

- 查看本月订阅总支出
- 设定月度预算并监控使用情况
- 接收续费提醒，避免意外扣费
- 管理已取消的订阅，避免重复订阅
- 统计分析订阅费用分布

## 3. 核心价值

### 3.1 清晰透明

直观展示订阅支出全貌，让用户清楚知道每月花费多少钱在订阅上。

### 3.2 预算控制

帮助用户设定月度预算，实时监控预算使用情况，超支时及时预警。

### 3.3 智能提醒

续费日期提前通知，避免意外扣费或错过续费。

### 3.4 省心省钱

减少订阅管理的认知负担，帮助用户识别不必要的订阅，节省开支。

## 4. 产品定位

**个人订阅管理工具 + 预算追踪助手**

- 不是记账软件，专注于订阅服务管理
- 不是提醒工具，而是全方位的订阅管理解决方案
- 支持本地存储和云端同步，满足不同用户需求

## 5. 技术架构概览

### 5.1 前端

- **平台**：微信小程序
- **框架**：weapp-vite + Vue 3
- **状态管理**：Pinia
- **本地存储**：微信小程序 Storage API

### 5.2 后端（云端方案）

- **云服务**：腾讯云 CloudBase
- **云托管**：CloudBase Run 运行 Go 后端容器服务
- **数据库**：MySQL
- **认证**：微信小程序原生认证（OPENID）

### 5.3 数据策略

- **云端为主**：数据存储在云端，支持多设备同步
- **本地缓存**：提升访问速度，支持离线查看
- **渐进式迁移**：从本地存储逐步迁移到云端

## 6. 竞品分析

### 6.1 市场现状

- 国外有 Truebill、Bobby 等订阅管理应用
- 国内缺乏专注于订阅管理的工具
- 现有记账软件功能过于复杂，不够聚焦

### 6.2 差异化优势

- **专注订阅**：不是通用记账工具，专注订阅管理
- **多币种支持**：支持多种货币，自动汇率转换
- **小程序形态**：无需下载 App，随时随地访问
- **本地优先**：数据隐私保护，支持完全本地运行
