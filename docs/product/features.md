# SubTrack 功能需求

## 1. 用户痛点分析

### 1.1 订阅支出模糊
用户不知道每月订阅总共花多少钱，缺乏整体支出概念。

### 1.2 忘记取消订阅
不再使用的服务仍在持续扣费，造成不必要的浪费。

### 1.3 续费日期难记
多个订阅的扣款日期分散，容易遗漏或忘记。

### 1.4 预算缺乏规划
没有设定订阅预算的概念，支出失控。

### 1.5 多币种困扰
不同订阅使用不同货币，难以统一计算总支出。

| 订阅状态管理 | 有效/已取消状态切换 | P1 |

### 2.2 预算管理（P0）

| 功能 | 描述 | 优先级 |
|------|------|--------|

| 未来费用预览 | 查看未来几个月预计支出 | P2 |

### 2.3 多币种支持（P0）

| 功能 | 描述 | 优先级 |
|------|------|--------|

### 2.4 数据可视化（P1）

| 功能 | 描述 | 优先级 |
|------|------|--------|

| 功能 | 描述 | 优先级 |
|------|------|--------|

## 3. 订阅信息字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 订阅名称 | String | 是 | 如：ChatGPT Plus, Notion |
| 金额 | Number | 是 | 订阅费用（单位：分） |
| 货币 | Enum | 是 | CNY/USD/EUR/GBP/JPY/HKD/TWD |
| 计费周期 | Enum | 是 | 月付/季付/年付 |
| 扣款日 | Number | 是 | 1-31 |
| 扣款月 | Number | 否 | 年付时的月份 1-12 |
| 开始日期 | Date | 否 | 订阅开始时间 |
| 备注 | String | 否 | 用途、账户信息等 |
| 状态 | Enum | 是 | 有效/已取消 |
| 取消日期 | Date | 否 | 取消订阅的日期 |
| 图标 | String | 否 | 订阅图标 URL |
| 分类 | String | 否 | AI工具/效率工具/娱乐等 |

## 4. 扣款日期规则

### 4.1 月付模式
每月 [1-31] 号扣款

**示例**：每月 15 号扣款

### 4.2 季付模式
每季度 [1-31] 号扣款（从开始日期算起，每 3 个月扣款一次）

**示例**：每季度 15 号扣款

### 4.3 年付模式
每年 [1-12] 月 [1-31] 日扣款

**示例**：每年 3 月 15 日扣款

## 5. 功能优先级说明

### P0 - 核心功能（MVP）
必须实现的功能，没有这些功能产品无法使用。

### P1 - 重要功能
显著提升用户体验的功能，第一版本应包含。

### P2 - 增值功能
锦上添花的功能，可在后续版本迭代。

### P3 - 未来规划
长期规划的功能，需要更多资源投入。

## 6. 非功能需求

### 6.1 性能要求
- 首屏加载时间 < 3 秒
- 页面切换流畅，无明显卡顿
- 列表滚动流畅（虚拟列表）

### 6.2 兼容性要求
- 支持微信小程序基础库 2.10.0+
- 适配 iOS 和 Android 平台

### 6.3 安全要求
- 用户数据加密存储
- API 请求使用 HTTPS
- 敏感信息不记录日志

### 6.4 可用性要求
- 支持离线查看（本地缓存）
- 网络异常时友好提示
- 数据同步失败时重试机制
