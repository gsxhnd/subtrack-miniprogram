## 1. 整体架构

│  └──────────┬───────────────┘   │
│             │                   │
│  ┌──────────▼───────────────┐   │
│  │   本地存储 (Storage)      │   │
│  └──────────────────────────┘   │
└─────────────────────────────────┘

**阶段 2：云端架构（规划中）**

│  ┌──────────────────────────────────┐   │

│  │   - Go 后端服务                   │   │
│  │   - 复杂业务逻辑                  │   │
│  └──────────┬───────────────────────┘   │

              ▼

## 2. 技术栈

### 2.1 前端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | weapp-vite + Vue 3 | 现代化小程序开发工具 |
| UI 组件 | 微信小程序原生组件 | 性能最优 |
| 状态管理 | Pinia | Vue 3 官方推荐 |
| 本地存储 | wx.storage | 微信小程序 API |
| 图表库 | Chart.js | 轻量级图表库 |

### 2.2 后端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 云平台 | 腾讯云 CloudBase | 一站式云服务 |
| 云函数 | Cloud Functions | 主要后端服务，自动获取 OPENID |
| 云托管 | Cloud Run | 复杂业务逻辑（可选） |
| 后端语言 | Go | 高性能、类型安全 |
| Web 框架 | Fiber | 高性能 Web 框架 |
| 数据库 | MySQL 8.0+ | 关系型数据库 |
| ORM | GORM | 数据库操作 |
| 认证 | CloudBase 原生认证 | 微信小程序 OPENID 自动注入 |

## 3. 数据流向

### 3.1 读取流程

```
用户请求
    │
    ▼
检查本地缓存
    │
    ├─ 有效缓存 ──► 返回数据
    │
    └─ 无缓存/过期
        │
        ▼
    请求云端 API
        │
        ▼
    更新本地缓存
        │
        ▼
    返回数据
```

### 3.2 写入流程

```
用户操作
    │
    ▼
更新本地缓存（乐观更新）
    │
    ▼
发送 API 请求
    │
    ├─ 成功 ──► 确认更新
    │
    └─ 失败 ──► 回滚本地缓存
                  │
                  ▼
              提示用户重试
```

## 4. 本地缓存策略

### 4.1 缓存内容

| 数据类型 | 缓存时长 | 说明 |
|---------|---------|------|
| 订阅列表 | 永久 | 离线可用 |
| 用户设置 | 永久 | 离线可用 |
| 统计数据 | 1 小时 | 定期刷新 |
| 汇率数据 | 24 小时 | 每日更新 |

### 4.2 缓存更新策略

- **写入时更新**：用户修改数据时立即更新缓存
- **定时刷新**：统计数据每小时刷新一次
- **手动刷新**：用户下拉刷新时强制更新
- **版本控制**：使用版本号检测数据变更

## 5. 离线支持

### 5.1 离线功能

- ✅ 查看订阅列表
- ✅ 查看订阅详情
- ✅ 查看统计数据（缓存）
- ❌ 添加/编辑订阅（需联网）
- ❌ 删除订阅（需联网）

### 5.2 离线提示

网络异常时显示友好提示，引导用户检查网络连接。

## 6. 数据同步机制

### 6.1 同步策略

- **云端为主**：数据存储在云端，本地作为缓存
- **增量同步**：只同步变更的数据
- **冲突解决**：服务端时间戳优先

### 6.2 同步时机

- 应用启动时
- 用户主动刷新时
- 数据修改后
- 后台切换到前台时
