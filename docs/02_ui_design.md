# SubTrack 界面设计文档

## 1. 页面导航结构

### 1.1 整体架构

```
┌───────────────────────────────────────────────────┐
│                 自定义 TabBar 导航                  │
├─────────┬─────────┬─────────┬─────────┬──────────┤
│  首页   │ 订阅列表 │  + 添加  │  统计   │   我的   │
│   🏠    │   📋    │  (突出)  │   📊    │   👤    │
└────┬────┴────┬────┴────┬────┴────┬────┴────┬─────┘
     │         │          │         │         │
     │         │          │         │         └── 我的（个人中心）
     │         │          │         │              ├── 订阅概览
     │         │          │         │              ├── 通用设置
     │         │          │         │              │    ├── 月度预算
     │         │          │         │              │    ├── 汇率设置
     │         │          │         │              │    └── 续费提醒
     │         │          │         │              ├── 数据管理
     │         │          │         │              │    └── 导入数据
     │         │          │         │              └── 关于
     │         │          │         │
     │         │          │         └── 统计页
     │         │          │              ├── 分类统计（饼图）
     │         │          │              ├── 货币分布
     │         │          │              └── 支出趋势（折线图）
     │         │          │
     │         │          └── 添加订阅（非 Tab 页）
     │         │              └── 跳转到订阅编辑页
     │         │
     │         └── 订阅列表页
     │              ├── 筛选与排序
     │              └── 订阅卡片列表
     │                   └── 点击进入详情
     │                        └── 编辑/删除
     │
      └── 首页仪表盘
           ├── 支出总览卡片
           ├── 费用统计卡片
           └── 即将续费列表
```

> **说明：** 中间的 `+` 按钮不是一个实际的 Tab 页面，而是一个操作入口，点击后通过 `wx.navigateTo` 跳转到订阅编辑页。需要使用微信小程序自定义 TabBar（`custom-tab-bar`）实现中间按钮的放大突出效果，并通过顶部凹陷曲线承托按钮视觉重心。

### 1.2 页面列表

**TabBar 页面（主包）：**

| 页面 | 路由 | TabBar 位置 | 说明 |
|------|------|-------------|------|
| 首页 | pages/index/index | 第1个 | 仪表盘，展示概览信息 |
| 订阅列表 | pages/subscription/index | 第2个 | 所有订阅列表 |
| 统计 | pages/statistics/index | 第4个 | 数据统计分析 |
| 我的 | pages/profile/index | 第5个 | 个人中心，设置与数据管理 |

> **注意：** 第3个位置为 `+` 添加按钮，不对应实际 Tab 页面。统计页需从子包 `packageStatistics` 迁移到主包。

**子包页面：**

| 页面 | 路由 | 所属子包 | 说明 |
|------|------|----------|------|
| 订阅详情 | packageSubscription/pages/detail/index | packageSubscription | 单个订阅详情 |
| 订阅编辑 | packageSubscription/pages/edit/index | packageSubscription | 添加/编辑订阅 |
| 预算设置 | packageSettings/pages/budget/index | packageSettings | 月度预算设置 |
| 汇率设置 | packageSettings/pages/exchange-rate/index | packageSettings | 手动设置汇率 |

**注意：** 以上路由为微信小程序页面路径，位于 `weapp/src/` 目录下。TabBar 页面必须放在主包 `pages/` 中。

## 2. 自定义 TabBar 设计

### 2.1 TabBar 布局

采用微信小程序自定义 TabBar（`custom-tab-bar`），中间放置突出的 `+` 添加按钮，并在顶部使用凹陷曲线制造承托感。

```
                    ┌─────┐
                    │  +  │  ← 圆形按钮，上浮约 20rpx
                    │     │     直径 100-120rpx
                    └─────┘     主题强调色背景
┌──────────┬───────╮    ╭───────┬──────────┬──────────┐
│   首页   │   订阅 │    │  统计  │   我的   │          │
│    🏠    │   📋   │    │   📊   │   👤    │          │
└──────────┴───────╯____╰───────┴──────────┴──────────┘
              ↑ 顶部采用类似贝塞尔曲线的凹口承托 `+` 按钮
```

### 2.2 TabBar 配置

**app.json 配置要点：**

```json
{
  "tabBar": {
    "custom": true,
    "list": [
      { "pagePath": "pages/index/index", "text": "首页" },
      { "pagePath": "pages/subscription/index", "text": "订阅" },
      { "pagePath": "pages/statistics/index", "text": "统计" },
      { "pagePath": "pages/profile/index", "text": "我的" }
    ]
  }
}
```

> **注意：** `tabBar.list` 中不包含 `+` 按钮，`+` 按钮在自定义 TabBar 组件中单独实现。

### 2.3 + 按钮设计规范

| 属性 | 值 | 说明 |
|------|-----|------|
| 形状 | 圆形 | 区别于其他方形图标 |
| 大小 | 100-120rpx | 约为常规 Tab 图标的 1.5-2 倍 |
| 上浮 | 向上偏移 20rpx | 突破 TabBar 上边界 |
| 凹口 | 顶部中心下凹 | 通过类似贝塞尔曲线的轮廓承托按钮 |
| 背景色 | 主题强调色（渐变蓝/绿） | 形成视觉焦点 |
| 图标 | 白色 `+` 号 | 高对比度，清晰可辨 |
| 点击行为 | `wx.navigateTo` 到订阅编辑页 | 非 Tab 切换，而是页面跳转 |
| 点击反馈 | 缩放动画（scale 0.9 → 1.0） | 提供触觉反馈 |

### 2.4 TabBar 交互说明

- 4 个常规 Tab 使用 `wx.switchTab` 切换
- 中间 `+` 按钮使用 `wx.navigateTo` 跳转到 `packageSubscription/pages/edit/index`
- 当前激活的 Tab 高亮显示（图标和文字变色）
- `+` 按钮始终保持突出样式，不参与 Tab 激活状态切换
- TabBar 顶部凹口与 `+` 按钮保持视觉中心对齐，避免像独立浮层悬空

```
│  ┌──────────────────────────────┐  │

│  └──────────────────────────────┘  │
```

```
│  ┌──────────────────────────────┐  │
│  │ 可选，如用途、账户信息        │  │

│                                    │
│            [删除订阅]              │
│                                    │
└────────────────────────────────────┘
```

### 5.2 表单字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 订阅名称 | 文本输入 | 是 | 如：ChatGPT Plus |
| 金额 | 数字输入 | 是 | 订阅费用 |
| 货币 | 下拉选择 | 是 | CNY/USD/EUR等 |
| 计费周期 | 单选 | 是 | 月付/季付/年付 |
| 扣款日期 | 复合选择 | 是 | 根据周期动态展示 |
| 开始日期 | 日期选择 | 否 | 订阅开始时间 |
| 分类 | 下拉选择 | 否 | AI工具/效率工具等 |
| 备注 | 文本域 | 否 | 备注信息 |

### 5.3 扣款日期选择逻辑

**月付模式：**
```
每月 [1-31] 号扣款
```

**季付模式：**
```
每季度 [1-31] 号扣款
（从开始日期算起，每3个月扣款一次）
```

**年付模式：**
```
每年 [1-12] 月 [1-31] 日扣款
```

### 8.2 页面区域说明

| 区域 | 说明 |
|------|------|
| 订阅概览 | 顶部卡片，展示活跃/已取消订阅数量、月度/年度支出汇总 |
| 通用设置 | 预算、汇率、提醒等核心设置项 |
| 数据管理 | 导入、清空当前账号数据 |
| 关于 | 版本信息与反馈入口 |

### 8.3 设置项说明

| 设置项 | 说明 |
|--------|------|
| 月度预算 | 设定每月订阅预算上限，跳转到预算设置页 |
| 汇率设置 | 手动输入各货币相对于基准货币的汇率，跳转到汇率设置页 |
| 续费提醒 | 开启/关闭续费提醒 |
| 提前天数 | 提前多少天提醒（1-7天） |
| 显示原币种 | 是否同时显示原始货币金额 |
| 导入数据 | 从 JSON 文件导入数据 |
| 清除所有数据 | 清空当前账号的云端数据（需二次确认） |

### 8.4 交互说明

- 订阅概览区域为纯展示，数据由云端接口返回并实时渲染
- 点击「月度预算」→ 跳转到 `packageSettings/pages/budget/index`
- 点击「汇率设置」→ 跳转到 `packageSettings/pages/exchange-rate/index`
- 点击「导入数据」→ 使用弹窗或跳转子页面
- 点击「清除所有数据」→ Modal 弹窗二次确认后调用云端删除能力

## 11. 通用组件设计

### 11.1 预算进度组件 (BudgetProgress)

```
┌────────────────────────────────────┐
│  本月支出                          │
│  ¥ 128.00 / ¥ 200.00              │
│  ████████░░░░░░░░ 64%             │
└────────────────────────────────────┘
```

**Props:**
- `spent`: 已支出金额
- `budget`: 预算金额
- `currency`: 货币符号

**状态颜色:**
- 使用率 < 80%: 绿色
- 80% ≤ 使用率 ≤ 100%: 黄色
- 使用率 > 100%: 红色

### 11.2 订阅卡片组件 (SubscriptionCard)

```
┌────────────────────────────────────┐
│ [图标]  订阅名称                   │
│         ¥金额/周期  ·  状态信息    │
└────────────────────────────────────┘
```

**Props:**
- `icon`: 图标URL
- `name`: 订阅名称
- `amount`: 金额
- `currency`: 货币
- `cycle`: 计费周期
- `status`: 状态
- `nextBillingDate`: 下次扣款日期
- `cancelledAt`: 取消日期

### 11.3 金额输入组件 (AmountInput)

```
┌────────────────────────────────────┐
│ ¥ [金额输入框]      [货币选择 ▼]   │
└────────────────────────────────────┘
```

**Props:**
- `value`: 金额
- `currency`: 货币代码
- `onChange`: 值变更回调

### 11.4 扣款日期选择组件 (BillingDatePicker)

```
月付模式:
┌────────────────────────────────────┐
│ 每月 [ 15 ] 号扣款                 │
└────────────────────────────────────┘

年付模式:
┌────────────────────────────────────┐
│ 每年 [ 3 ] 月 [ 15 ] 日扣款        │
└────────────────────────────────────┘
```

**Props:**
- `cycle`: 计费周期
- `day`: 扣款日
- `month`: 扣款月份（年付时）
- `onChange`: 值变更回调
