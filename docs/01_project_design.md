# SubTrack 订阅预算追踪小程序 - 产品设计文档

## 1. 项目概述

### 1.1 项目背景

SubTrack 是一款个人财务管理类小程序，专注于追踪和管理用户的软件订阅服务。核心目标是帮助用户清晰了解每月/每年的订阅支出，并有效设定和控制预算。

### 1.2 目标用户

- 拥有多个软件/服务订阅的个人用户
- 希望管理订阅支出、控制预算的用户
- 经常忘记取消订阅或忘记续费日期的用户

### 1.3 核心价值

- **清晰透明**：直观展示订阅支出全貌
- **预算控制**：帮助用户设定和监控预算
- **智能提醒**：续费日期提前通知，避免意外扣费
- **省心省钱**：减少订阅管理的认知负担

### 1.4 数据与账号策略

- 业务数据统一存储在 CloudBase 云端，不再以本地存储作为主数据源。
- 用户身份通过微信小程序接入，使用可信 `openid` 关联个人数据。
- 页面展示的数据以云端返回结果为准，支持后续多端一致性扩展。

## 2. 需求分析

### 2.1 用户痛点

1. **订阅支出模糊**：不知道每月订阅总共花多少钱
2. **忘记取消订阅**：不再使用的服务仍在持续扣费
3. **续费日期难记**：多个订阅的扣款日期分散，容易遗漏
4. **预算缺乏规划**：没有设定订阅预算的概念
5. **多币种困扰**：不同订阅使用不同货币，难以统一计算

### 2.2 功能需求

#### 2.2.1 订阅记录管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 添加订阅 | 创建新的订阅记录 | P0 |
| 编辑订阅 | 修改订阅信息 | P0 |
| 删除订阅 | 移除订阅记录 | P0 |
| 状态管理 | 标记订阅为"有效"或"已取消" | P1 |
| 取消订阅 | 记录取消日期 | P1 |

#### 2.2.2 预算管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 月度预算设置 | 设定每月订阅预算上限 | P0 |
| 支出计算 | 自动计算本月预计支出 | P0 |
| 多币种支持 | 支持人民币、美元等多种货币 | P0 |
| 预算进度展示 | 可视化预算使用情况 | P0 |
| 未来费用预览 | 查看未来几个月预计支出 | P1 |

#### 2.2.3 用户体验

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 首页仪表盘 | 总支出、预算进度、即将续费列表 | P0 |
| 订阅列表页 | 列表/卡片展示，支持排序筛选 | P0 |
| 数据可视化 | 饼图展示费用分布 | P1 |
| 续费提醒 | 订阅消息提醒（需授权） | P2 |

## 3. 功能设计

### 3.1 订阅信息字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 订阅名称 | String | 是 | 如：ChatGPT Plus, Notion |
| 金额 | Number | 是 | 订阅费用 |
| 货币 | Enum | 是 | CNY/USD/EUR/GBP/JPY/HKD/TWD |
| 计费周期 | Enum | 是 | 月付/季付/年付 |
| 扣款日期规则 | Object | 是 | 用户自定义扣款日期 |
| 开始日期 | Date | 否 | 订阅开始时间 |
| 备注 | String | 否 | 用途、账户信息等 |
| 状态 | Enum | 是 | 有效/已取消 |
| 取消日期 | Date | 否 | 取消订阅的日期 |
| 图标 | String | 否 | 订阅图标URL |
| 分类 | String | 否 | 订阅分类 |

### 3.2 扣款日期规则

用户可以自定义扣款日期，支持以下模式：

#### 月付模式

```
每月 [1-28] 号扣款
```

示例：每月15号扣款

#### 季付模式

```
每季度 [1-28] 号扣款
（从开始日期算起，每3个月扣款一次）
```

示例：每季度15号扣款

#### 年付模式

```
每年 [1-12] 月 [1-28] 日扣款
```

示例：每年3月15日扣款

### 3.3 多币种支持

#### 支持的货币

| 货币代码 | 货币名称 | 符号 |
|----------|----------|------|
| CNY | 人民币 | ¥ |
| USD | 美元 | $ |
| EUR | 欧元 | € |
| GBP | 英镑 | £ |
| JPY | 日元 | ¥ |
| HKD | 港币 | HK$ |
| TWD | 新台币 | NT$ |

#### 汇率处理

- 使用实时汇率API获取汇率数据
- 汇率数据缓存1小时
- 用户可设置基准货币（默认CNY）
- 支持同时显示原币种和转换后金额

### 3.4 费用计算规则

#### 月度支出计算

```
本月支出 = Σ(有效订阅的月均费用，转换为基准货币)

月均费用计算：
- 月付：金额
- 季付：金额 / 3
- 年付：金额 / 12
```

#### 预算进度计算

```
预算使用率 = 本月支出 / 月度预算 × 100%

颜色标识：
- < 80%：绿色（正常）
- 80% - 100%：黄色（警告）
- > 100%：红色（超支）
```

### 3.5 订阅状态管理

#### 状态定义

| 状态 | 说明 |
|------|------|
| 有效 (active) | 当前正在使用的订阅 |
| 已取消 (cancelled) | 已取消的订阅，记录取消日期 |

#### 取消后重新订阅的处理策略

**方案：创建新的订阅记录**

当用户取消订阅后重新续费时：
1. 原订阅记录保持"已取消"状态，保留取消日期
2. 创建一条新的订阅记录
3. 历史记录完整保留，便于统计分析

```
创建订阅 ──────────────► 有效 (active)
    │                        │
    │                        ▼
    │                   取消订阅
    │                        │
    │                        ▼
    │                   已取消 (cancelled)
    │                   记录取消日期
    │                        │
    │                        ▼
    │                   重新续费
    │                        │
    │                        ▼
    └──────────────────► 创建新订阅记录 (active)
```

### 3.6 续费提醒规则

- 默认提前3天提醒
- 用户可自定义提前天数（1-7天）
- 需用户授权订阅消息
- 提醒内容：订阅名称、金额、货币、扣款日期

| 字段名 | 类型 | 说明 |

| year | Number | 年份 |
